# 测试指南

## 浏览器测试

### 推荐测试的浏览器
- **Chrome** (最新版本)
- **Firefox** (最新版本)
- **Safari** (最新版本)
- **Edge** (最新版本)

### 测试步骤
1. 在浏览器中打开 `index.html`
2. 检查页面是否正常加载
3. 验证作品网格是否正确显示
4. 测试分类过滤功能
5. 点击作品卡片，验证是否跳转到正确页面
6. 测试返回按钮和导航

## 移动端测试

### 测试设备
- 智能手机（iPhone/Android）
- 平板电脑（iPad/Android平板）

### 测试项目
1. 响应式布局是否正常工作
2. 触摸目标大小是否合适（最小44×44像素）
3. 图片是否适配不同屏幕尺寸
4. 文本大小是否可读

## 可访问性测试

### 键盘导航
1. 使用Tab键浏览所有交互元素
2. 验证焦点指示器是否可见
3. 测试所有功能是否可通过键盘操作

### 屏幕阅读器测试
1. 图片是否都有适当的alt文本
2. 语义化HTML结构是否正确
3. 音频内容是否有文字转录

### 颜色对比度
使用工具检查文本与背景的对比度是否达到WCAG AA标准（4.5:1）

## 功能测试

### 主页功能
- [ ] 作品网格正确显示所有作品
- [ ] 分类过滤功能正常工作
- [ ] 点击作品卡片跳转到正确页面
- [ ] 页面标题和元信息正确

### 作品详情页面
- [ ] 作品信息正确显示
- [ ] 图片加载和显示正常
- [ ] 音频播放器功能正常
- [ ] 图片灯箱/查看器功能正常
- [ ] 返回导航正常工作

### 错误处理
- [ ] 作品数据加载失败时显示适当错误信息
- [ ] 图片加载失败时显示占位图片
- [ ] 404页面正常工作

## 性能测试

### 页面加载速度
1. 首次加载时间
2. 图片懒加载效果
3. 缓存策略

### 优化建议
1. 压缩图片文件
2. 使用现代图片格式（WebP）
3. 最小化CSS和JavaScript文件

## 部署测试

### 本地测试
1. 在本地文件系统测试
2. 使用本地HTTP服务器测试（如 `python -m http.server`）

### 生产环境测试
1. 部署到测试服务器
2. 验证所有URL和路径正确
3. 测试跨域资源加载

## 自动化测试

### 简单测试脚本
```bash
# 检查文件结构
find . -name "*.html" -type f | wc -l
find . -name "*.css" -type f | wc -l
find . -name "*.js" -type f | wc -l

# 检查JSON文件有效性
node -e "const fs=require('fs'); try { JSON.parse(fs.readFileSync('data/artworks.json')); console.log('artworks.json valid') } catch(e) { console.error('artworks.json invalid:', e.message) }"
```

## 问题排查

### 常见问题
1. **图片不显示**：检查文件路径和权限
2. **JavaScript错误**：检查浏览器控制台
3. **样式问题**：检查CSS文件加载
4. **音频不播放**：检查文件格式和路径

### 调试工具
- 浏览器开发者工具
- 网络面板检查资源加载
- 控制台查看错误信息
- 移动端模拟器

## 维护测试

### 添加新作品后测试
1. 验证新作品出现在主页网格
2. 测试作品详情页面
3. 检查分类计数更新

### 更新样式后测试
1. 跨浏览器一致性测试
2. 移动端响应式测试
3. 可访问性影响评估
